## key如何设置过期时间？

执行命令

```
SET key value NX PX 1000 # 设置1000ms后过期
PEXPIRE key 2000 # 设置2000ms后过期
TTL key # 数字代表过期剩余时间，-1代表不过期，-2代表已过期
```



## redis如何判定key已过期？

对key设置过期时间时，redis会把key放入过期字典，访问key时，通过过期字典判断key是否已过期

- 过期字典数据结构：哈希表，key是指针，指向某个键对象，value是long long类型整数，记录key过期时间
- 过程1：对key设置过期时间时，redis会把key放入过期字典
- 过程2：查询key时，redis检查key在不在过期字典中，如果不在，返回键值，如果在，将过期时间和系统时间比对，如果比系统时间小（在系统时间之前），判定key已过期



## 过期删除策略有哪些？

定时删除

- 做法：在设置key过期时间时，创建一个定时事件，当到达过期时间时，执行删除key操作
- 优点：保证key会被马上删除
- 缺点：过期key较多时，对CPU消耗大

惰性删除

- 做法：不主动删除过期key，只有key被访问时，才检查key是否过期，如果过期则删除
- 优点：对CPU消耗小
- 缺点：如果key一直不被访问，key无法被删除，造成内存浪费

定期删除

- 做法：每隔一段时间，【随机】抽一批key检查，删除过期的key
- 优点：对CPU消耗可控，和惰性删除相比，能删除较多的过期key
- 缺点：很难确定删除操作的频率，如果太频繁，和定时删除一样，对CPU消耗大；如果频率太小，和惰性删除一样，造成内存浪费



## redis过期删除策略是什么？key过期会不会立刻删除？

惰性删除 + 定期删除

- 惰性删除：不主动删除过期key，只有key被访问时，才检查key是否过期，如果过期则删除
- 定期删除：redis每隔一段时间，【随机】抽一批key检查，删除过期的key。（默认每秒10次），每次抽取20个key，如果过期key比例大于25%，就重复这个过程，（直到超过25ms超时时间）



## 什么是内存淘汰策略？

redis内存超过上限值后，会使用内存淘汰策略，删除符合条件的key

- 查看配置

  ```
  CONFIG GET maxmemory
  ```



## redis内存淘汰策略有哪些？

redis内存淘汰策略主要有三种

【不进行数据淘汰】

- noeviction（默认），超过最大内存时，不淘汰数据。写入新数据报错，读取数据没有影响

【在设置了过期时间的键值中，进行淘汰】

- volatile-random，在设置了过期时间的键值中，随机淘汰
- volatile-ttl，在设置了过期时间的键值中，淘汰最早过期的键值
- volatile-lru，在设置了过期时间的键值中，淘汰最久未使用的键值
- volatile-lfu（Redis4.0后），在设置了过期时间的键值中，淘汰最少使用的键值

【在所有键值中，进行淘汰】

- allkeys-random，在所有键值中，随机淘汰
- allkeys-lru，在所有键值中，淘汰最久未使用的键值
- allkeys-lfu（Redis4.0后），在所有键值中，淘汰最少使用的键值



## 什么是传统LRU（Least Recently Used）？

是一种内存淘汰策略，淘汰最久未使用的键值

- 做法：基于链表实现，链头是最近被使用的键值，链尾是最久未使用的键值，键值被访问时，移动对应的节点到链头，进行内存淘汰时，删除链尾节点



## 什么是redis的近似LRU？

redis的LRU和传统LRU不同，进行内存淘汰时，随机选5个键值，淘汰最久未使用键值

- 做法：在redis对象中，加入字段lru表示【最后一次访问时间】，进行内存淘汰时，随机挑5个key，淘汰最久未使用键值
- 好处：不用维护链表，也不用移动节点，节省时间空间
- 缺点：只使用过一次的键值，要存活很久才能被删除





## 什么是传统LFU（Least Frequently Used）？

是一种内存淘汰策略，淘汰最少使用的键值

- 做法，LFU记录了每个数据的访问次数，进行内存淘汰时，淘汰最少使用的键值



## 什么是redis的近似LFU？

在redis对象中，复用字段lru表示【访问时间戳 + 频率】，访问键值时，对频率做衰减和增加操作，淘汰最少使用的键值

- 做法，在redis对象中，复用字段lru，前16位表示访问时间戳，后8位是访问频率
- 频率变化，频率默认是5，键值被访问时，对频率做衰减操作，本次访问时间和上次间隔越久，衰减越多，再对频率做增加操作，频率越大增加幅度越小，最大增加至25
- 查看配置

```
CONFIG GET lfu-decay-time # 控制logc的衰减速度，默认是1，表示xxx分钟衰减一次
CONFIG GET lfu-log-factor # 控制logc的增长速度，默认是10，值越大，增长越慢
```



## 过期删除和内存淘汰有什么区别？

触发条件不一样，过期删除是发现键值过期时，删除键值；内存淘汰是redis内存超过上限值后，删除键值
