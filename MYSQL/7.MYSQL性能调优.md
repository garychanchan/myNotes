## explain的作用？

explain是查看sql的执行计划，分析sql的执行过程，判断sql有没有使用索引、外部排序、索引覆盖等等



## explain执行计划参数？

explain执行计划参数

- possible_keys，可能使用的索引

- key，实际使用的索引，如果为NULL，表示没有使用索引

- key_len，索引的长度

- rows，扫描的行数

- type，数据扫描类型

  

type数据扫描类型参数（执行效率从低到高排序）

- all，全表扫描，发生在没用索引，遍历所有行获得结果

  ```mysql
  SELECT * FROM tbl_user
  SELECT * FROM tbl_user WHERE age > 30
  ```

- index，全索引扫描，发生在select字段是主键或索引，只查询索引树就可以获得结果

  ```mysql
  SELECT id FROM tbl_user -- 聚簇索引
  SELECT money FROM tbl_user -- 非聚簇索引
  ```

- range，索引范围扫描，发生在索引树范围查询

  ```mysql
  SELECT * FROM tbl_user WHERE id > 30 -- 聚簇索引
  SELECT * FROM tbl_user WHERE money > 30 -- 非聚簇索引
  ```

- ref，非唯一索引扫描，发生在非聚簇索引等值查询

  ```mysql
  SELECT * FROM tbl_user WHERE money = 30
  ```

- eq_ref，唯一索引扫描，通常发生在关联查询，关联查询的条件是主键或UNIQUE

  ```
  xxx
  ```

- const，对主键或UNIQUE唯一索引的等值查询，最多返回一行数据

  ```mysql
  SELECT * FROM tbl_user WHERE id = 1
  SELECT * FROM tbl_user WHERE `name` = 'gary'
  ```

- system，const的特殊情况，表里只有一条数据

  ```
  xxx
  ```

  

extra参数

- Using filesort，使用了文件排序，语句包含group by，且无法利用索引完成排序操作

- Using temporary，使用了临时表保存中间结果

- Using index，索引覆盖

  ```mysql
  SELECT money FROM tbl_user
  ```

- Using index condition，索引条件下推，MYSQL存储引擎过滤数据

  ```mysql
  SELECT * FROM tbl_user WHERE money > 30 -- 非聚簇索引
  ```

- Using where，MYSQL服务器层过滤数据



## 怎么看语句是否使用索引？

使用EXPLAIN查看SQL语句的执行计划，看key字段是否有值



## 查看表的索引？

执行SHOW INDEX命令，或执行SHOW KEYS命令

```mysql
SHOW INDEX FROM tbl_user
SELECT * FROM information_schema.STATISTICS WHERE TABLE_SCHEMA = 'gary' AND TABLE_NAME = 'tbl_user'
```



## 给你张表，查询速度很慢，有哪些解决方案？

查看慢SQL的执行计划，分析是SQL语句还是表的问题。如果是SQL语句，检查索引是否失效，优化SQL语句（使用索引覆盖和索引下推技术）；如果是表的问题，检查索引是否设置合理（高频字段设置为索引），表数据量是否过大（拆表）。使用redis存放高频查询数据，减轻MYSQL压力



- 分析SQL语句执行计划，使用EXPLAIN命令查看SQL语句的执行计划，看看是不是进行了全表扫描，或没使用索引
- 避免索引失效，查询结果过多、LIKE以通配符开头、对索引使用函数等
- 查询优化，避免使用SELECT *，使用索引覆盖和索引下推技术
- 优化索引，经常用于WHERE、ORDER BY、JOIN、GROUP BY的字段，设置索引
- 优化数据库表，数据量超千万，将大表拆分为小表
- 使用缓存，使用redis存放高频查询数据，减轻MYSQL压力
